## ② 우리동네 모드

사용하는 스크립트: `scripts\tour_skill.py`

### 수업 시작 흐름

#### 단계 1: Nominatim 지오코딩 (WebFetch GET)

```
https://nominatim.openstreetmap.org/search?q={입력값}&format=json&limit=1&accept-language=ko
```

추출: `lat`, `lon` (기준점 좌표)

#### 단계 2: Overpass — 공공시설 검색 (WebFetch GET)

학교 제외, 공공시설만, `around:2000` (2km 반경):

```
https://overpass-api.de/api/interpreter?data=[out:json][timeout:30];(node(around:2000,{lat},{lon})["amenity"="community_centre"];way(around:2000,{lat},{lon})["amenity"="community_centre"];node(around:2000,{lat},{lon})["office"="government"];way(around:2000,{lat},{lon})["office"="government"];node(around:2000,{lat},{lon})["amenity"="townhall"];way(around:2000,{lat},{lon})["amenity"="townhall"];node(around:2000,{lat},{lon})["amenity"="police"];way(around:2000,{lat},{lon})["amenity"="police"];node(around:2000,{lat},{lon})["amenity"="fire_station"];way(around:2000,{lat},{lon})["amenity"="fire_station"];node(around:2000,{lat},{lon})["amenity"="library"];way(around:2000,{lat},{lon})["amenity"="library"];node(around:2000,{lat},{lon})["amenity"="post_office"];way(around:2000,{lat},{lon})["amenity"="post_office"];);out center;
```

결과 3개 미만이면 반경 3000으로 재시도.

유형 분류:
| Overpass 태그 | 내부 키 | 뱃지 |
|---|---|---|
| `amenity=community_centre` | `community` | 🏘️ |
| `office=government`, `amenity=townhall` | `government` | 🏛️ |
| `amenity=police` | `police` | 👮 |
| `amenity=fire_station` | `fire` | 🚒 |
| `amenity=library` | `library` | 📚 |
| `amenity=post_office` | `post_office` | 📮 |

- 좌표: node → `element.lat/lon`, way → `element.center.lat/center.lon`
- 이름: `tags.name` → `tags["name:ko"]` → `tags.operator` → 유형기본명
- 유형별 최대 3개

#### 단계 3: 시설별 Wikimedia 사진 검색 (선택)

"사진 없이 시작" 입력 시 건너뜀.

```
https://commons.wikimedia.org/w/api.php?action=query&list=search&srsearch={시설명}&srnamespace=6&format=json&srlimit=1
```
→ `search[0].title` 파일명 추출  
→ 실패 시 `{도시명} {시설유형}`으로 재검색  
→ 썸네일 URL 획득:
```
https://commons.wikimedia.org/w/api.php?action=query&titles={파일명}&prop=imageinfo&iiprop=url&iiurlwidth=400&format=json
```

#### 단계 4: locations JSON 구성

```json
{
  "locations": {
    "community":   [{"name":"...", "lat":37.x, "lon":126.x, "photo_url":"..."}],
    "government":  [],
    "police":      [],
    "fire":        [],
    "library":     [],
    "post_office": []
  }
}
```

#### 단계 5: neighborhood 커맨드 실행 (PowerShell)

```powershell
$env:PYTHONUTF8 = "1"
python "scripts\tour_skill.py" neighborhood `
  --region "{학교명}" `
  --center-lat {lat} --center-lon {lon} `
  --radius 2.0 `
  --data '{JSON 문자열}'
```

#### 단계 6: 서버 시작 + 탐방 시작

```powershell
$env:PYTHONUTF8 = "1"
$listening = netstat -an 2>$null | Select-String "127\.0\.0\.1:8765.*LISTEN"
if (-not $listening) {
    Start-Process python -ArgumentList "`"scripts\tour_skill.py`" serve" -WindowStyle Hidden
    Start-Sleep -Milliseconds 900
}
python "scripts\tour_skill.py" tour-nav --action start
Start-Process "http://localhost:8765"
```

#### 수업 중 명령어

| 명령 | 처리 |
|------|------|
| `다음` | `tour-nav --action next` → 시설 정보 출력 |
| `이전` | `tour-nav --action prev` → 시설 정보 출력 |
| `퀴즈` | 현재 시설 `student_questions[0]`을 4지선다로 출력 |
| `활동지` | `worksheet` 커맨드 |
| `보고서` | `report` 커맨드 |

#### 시설 정보 출력 형식

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
{뱃지} {index}/{total}: {name}  ({distance_km}km)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📌 위치: {lat}°N, {lon}°E

🏙️ 시설 안내:
{historical_significance}

👁️ 관찰 포인트:
1. ...

❓ 탐구 질문:
1. ...

{'다음'을 입력하면 다음 시설로 이동합니다. / 🏁 탐방 완료!}
```

## ③ 지형등고선 모드

사용 스크립트: `scripts\tour_skill.py`

### 데이터 소스
- **육상 지형** (산, 고원, 계곡): SRTM 90m — `srtm90m` 데이터셋
- **해양 지형** (해령, 해구, 해저 분지): ETOPO1 — `etopo1` 데이터셋 (음수 수심 포함)

### 수업 시작 흐름

#### 단계 1: 지형명 지오코딩 (WebFetch Nominatim)

```
GET https://nominatim.openstreetmap.org/search?q={지형명}&format=json&limit=1&accept-language=ko
```

추출: `lat`, `lon`

해양 지형(해령·해구)은 Nominatim에 없을 수 있음 → 아래 좌표를 사용:

| 지형 | lat | lon | radius_km |
|------|-----|-----|-----------|
| 마리아나 해구 | 11.35 | 142.2 | 300 |
| 대서양 중앙 해령 | 0.0 | -25.0 | 400 |
| 동태평양 해팽 | -15.0 | -110.0 | 300 |
| 일본 해구 | 39.0 | 143.5 | 200 |
| 페루·칠레 해구 | -23.0 | -71.0 | 200 |
| 수마트라 해구 | 5.0 | 94.0 | 200 |

#### 단계 2: 반경 결정

| 지형 유형 | 권장 radius_km |
|-----------|---------------|
| 일반 산봉우리 (한라산·설악산 등) | 15~25 |
| 큰 산맥 (히말라야·알프스) | 50~100 |
| 고원·분지 | 50~150 |
| 해령 (mid-ocean ridge) | 200~500 |
| 해구 (ocean trench) | 150~300 |

#### 단계 3: terrain 커맨드 실행 (PowerShell)

육상 지형:
```powershell
$env:PYTHONUTF8="1"
& python "scripts\tour_skill.py" terrain `
  --name "{지형명}" --lat {lat} --lon {lon} --radius {radius_km}
```

해양 지형 (`--ocean` 플래그 추가):
```powershell
$env:PYTHONUTF8="1"
& python "scripts\tour_skill.py" terrain `
  --name "{지형명}" --lat {lat} --lon {lon} --radius {radius_km} --ocean
```

#### 단계 4: 서버 시작 + 브라우저 열기

```powershell
$env:PYTHONUTF8="1"
$listening = netstat -an 2>$null | Select-String "127\.0\.0\.1:8765.*LISTEN"
if (-not $listening) {
    Start-Process python -ArgumentList "`"scripts\tour_skill.py`" serve" -WindowStyle Hidden
    Start-Sleep -Milliseconds 900
}
Start-Process "http://localhost:8765/terrain"
```

#### 단계 5: 결과 출력 형식

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⛰️ {지형명} 등고선 지도 로딩 중
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📌 위치: {lat}°N, {lon}°E
📐 반경: {radius_km}km
🗂️ 데이터: {srtm90m / etopo1}

브라우저에서 http://localhost:8765/terrain 가 열립니다.

왼쪽 패널:
  🗺️ [등고선 탭] — 위에서 본 등고선 지도 (색상 + 마칭스퀘어 등고선)
  ⛰️ [3D 탭]    — Three.js 3D 지형 모델 (마우스 드래그로 회전)
  📊 하단 단면도  — 동서 고도/수심 프로파일

오른쪽 패널:
  🛰️ 위성 지도 + 🌏 Google Earth 버튼

고도 데이터 로딩에 20~40초가 소요됩니다 (ETOPO1/SRTM API 조회).
```

## ⑤ 한랭지대 탐방 모드

### 포함 지역 (6곳)
오이먀콘(시베리아) → 알래스카 → 캐나다 북부 → 스칸디나비아 북부 → 그린란드 → 남극 대륙

### 실행 (PowerShell)

```powershell
$env:PYTHONUTF8="1"
$listening = netstat -an 2>$null | Select-String "127\.0\.0\.1:8765.*LISTEN"
if (-not $listening) {
    Start-Process python -ArgumentList "`"scripts\tour_skill.py`" serve" -WindowStyle Hidden
    Start-Sleep -Milliseconds 900
}
& python "scripts\tour_skill.py" climate --zone cold
Start-Process "http://localhost:8765/climate"
```

## ⑥ 기후지대 비교 지도 모드

### 실행 (PowerShell)

```powershell
$env:PYTHONUTF8="1"
$listening = netstat -an 2>$null | Select-String "127\.0\.0\.1:8765.*LISTEN"
if (-not $listening) {
    Start-Process python -ArgumentList "`"scripts\tour_skill.py`" serve" -WindowStyle Hidden
    Start-Sleep -Milliseconds 900
}
Start-Process "http://localhost:8765/climate-map"
```

### 화면 구성
- 전체 화면 위성 세계 지도 (줌 레벨 2)
- 굵은 주요 위도선:
  - 🟡 적도 (0°) — 굵기 4, 황금색
  - 🟠 북·남회귀선 (±23.5°) — 굵기 3, 주황 점선
  - 🔵 북·남극권 (±66.5°) — 굵기 3, 하늘색 점선
- 일반 격자선: 30° 간격 위·경도선 (반투명 흰색)
- 🟧 건조지대: 주황 반투명 사각형 (10개 지역)
- 🟦 한랭지대: 파랑 반투명 사각형 (8개 지역)
- 마우스 올리면 지역명 툴팁
- 우하단 범례 패널
